# HTTP/3.0
  - 2.0 的缺陷：
    1. TCP的队头阻塞：因为TCP的传输是有序的，单个数据包丢失会造成阻塞
    2. TCP建立连接的延时: TCP握手，HTTPS中的TLS加密过程也有一次握手

  - 改进TCP协议
    TCP僵化，实现不可能
    1. 太多中间设备基于TCP生产，路由器，交换机...
    2. 操作系统也是内核也是基于TCP来工作

  - QUIC协议 (UDP升级)
    1. 有多路复用
    2. 传输可靠

  - 面临的挑战
    1. 浏览器和服务器都要升级支持QUIC协议
    2. 系统对UDP的优化很低，对TCP的优化很高
    3. 中间设备僵化
